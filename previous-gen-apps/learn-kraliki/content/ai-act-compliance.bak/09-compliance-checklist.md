# Compliance Checklist

## Complete AI Act Compliance Checklist

Use this checklist to verify your organization's AI Act compliance.

## Part 1: Initial Assessment

### AI System Inventory
- [ ] List all AI systems in use
- [ ] Identify system purpose and intended use
- [ ] Document system architecture
- [ ] Map data flows
- [ ] Identify all stakeholders

### Risk Classification
- [ ] Classify each AI system by risk category:
  - [ ] Prohibited (none allowed)
  - [ ] High-risk (requires certification)
  - [ ] Limited-risk (transparency only)
  - [ ] Minimal-risk (no specific requirements)

### Regulatory Deadlines
- [ ] Identify applicable compliance deadlines
- [ ] Prioritize systems by deadline
- [ ] Create compliance roadmap
- [ ] Allocate resources

## Part 2: Prohibited AI Verification

### Immediate Action
- [ ] **No prohibited AI systems in use**
  - [ ] No subliminal manipulation
  - [ ] No exploitation of vulnerabilities
  - [ ] No social scoring
  - [ ] No real-time biometric ID in public
  - [ ] No predictive policing
  - [ ] No biometric categorization of protected characteristics
  - [ ] No emotion recognition in workplaces/education
  - [ ] No unauthorized facial image scraping

### Transition (if applicable)
- [ ] Prohibited systems shut down
- [ ] Shutdown documented
- [ ] Affected users notified
- [ ] Alternatives identified
- [ ] Compliance verified

## Part 3: High-Risk AI Requirements

### 1. Risk Management System
- [ ] Risk management process documented
- [ ] Risk register created
- [ ] Mitigation strategies defined
- [ ] Regular review schedule established (quarterly)
- [ ] Continuous risk monitoring in place

### 2. High-Quality Data
- [ ] Data quality assessment completed
- [ ] Bias detection performed
- [ ] Bias mitigation implemented
- [ ] Data governance procedures documented
- [ ] Data sources and provenance documented
- [ ] Legal basis for data use verified (GDPR)

### 3. Technical Documentation
- [ ] General system information
- [ ] System architecture diagrams
- [ ] Data flow documentation
- [ ] Risk assessment documentation
- [ ] Quality management procedures
- [ ] Instructions for use
- [ ] Monitoring mechanisms documented

### 4. Record-Keeping
- [ ] Automatic logging configured
- [ ] System inputs/outputs logged
- [ ] Timestamps recorded
- [ ] User identification captured
- [ ] Decision outcomes logged
- [ ] 10-year retention configured
- [ ] Log access controls implemented

### 5. Transparency & User Information
- [ ] System purpose communicated
- [ ] Accuracy and robustness disclosed
- [ ] Limitations and restrictions documented
- [ ] Human oversight mechanisms explained
- [ ] Monitoring procedures communicated

### 6. Human Oversight
- [ ] Human oversight policies defined
- [ ] Override mechanisms implemented
- [ ] Human review periods established
- [ ] Human operators trained
- [ ] Accountability structure defined

### 7. Accuracy, Robustness, Cybersecurity
- [ ] Accuracy validated
- [ ] Confidence scores provided
- [ ] Error rates documented
- [ ] Robustness testing completed
- [ ] Adversarial resistance tested
- [ ] Security controls implemented
- [ ] Access controls configured
- [ ] Security audits completed
- [ ] Incident response procedures documented

### 8. Conformity Assessment
- [ ] Self-assessment completed
- [ ] Notified body selected
- [ ] Third-party assessment initiated
- [ ] Declaration of conformity prepared
- [ ] CE marking applied (if required)

### 9. Quality Management System
- [ ] Quality policy documented
- [ ] Quality objectives defined
- [ ] Development procedures documented
- [ ] Testing protocols defined
- [ ] Document control system implemented
- [ ] Supplier management procedures
- [ ] Continuous improvement process

### 10. Post-Market Monitoring
- [ ] Performance metrics defined
- [ ] Monitoring dashboard configured
- [ ] User feedback system implemented
- [ ] Incident reporting process established
- [ ] Regular audit schedule set
- [ ] Risk reassessment process defined

## Part 4: Limited-Risk AI Requirements

### Chatbots
- [ ] AI disclosure clearly displayed
- [ ] Human operator option available
- [ ] No impersonation of humans

### AI-Generated Content
- [ ] Content labeled as AI-generated
- [ ] Watermarks applied (if applicable)
- [ ] Metadata tags included

### Emotion Recognition
- [ ] Users informed before analysis
- [ ] Clear consent obtained
- [ ] Opt-out options available
- [ ] Data retention policies defined

### Biometric Categorization
- [ ] Categories disclosed
- [ ] Purpose explained
- [ ] Data rights information provided
